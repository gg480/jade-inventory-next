## 1. 系统部署与首次登录

### 1.1 部署系统

**Docker 方式（推荐）**：
1. 确保服务器已安装 Docker
2. 执行部署命令：
   ```bash
   docker run -d \
     --name jade-inventory \
     -p 3000:3000 \
     -e ADMIN_PASSWORD=你的安全密码 \
     -v jade-db:/app/db \
     -v jade-uploads:/app/upload \
     --restart unless-stopped \
     gg480/jade-inventory-next:latest
   ```
3. 浏览器访问 `http://服务器IP:3000`

> 首次启动时系统会自动初始化数据库和基础数据（材质字典、器型字典、标签字典）。

### 1.2 登录系统

1. 打开系统页面，显示登录界面
2. 输入管理员密码（默认 `admin123`，如通过环境变量设置则使用设置的密码）
3. 点击"登录"按钮
4. 登录成功后自动跳转到利润看板

### 1.3 修改密码

1. 点击底部"退出登录"旁的设置图标，或切换到"系统设置"Tab
2. 找到"修改密码"区域
3. 输入当前密码和新密码（至少6位）
4. 点击确认，密码即时生效

## 2. 核心工作流程

### 2.1 整体业务流程

```
采购入库 → 库存管理 → 销售出库 → 客户管理
    │          │          │          │
  批次分摊   扫码开单   支付跟踪   VIP分层
    │          │          │          │
  成本核算   标签打印   退货处理   偏好分析
                              │
                         利润看板 (全链路数据分析)
```

### 2.2 高货入库流程（单件贵重货品）

**适用场景**：单件高价值翡翠、和田玉等货品入库

1. 切换到 **库存管理** Tab
2. 点击右上角 **"新增货品"** 按钮
3. 填写货品信息：
   - **材质选择**：先选大类（玉/贵金属/水晶/文玩/其他）→ 子类 → 具体材质
   - **器型选择**：手镯/挂件/吊坠/手串/项链/脚链/戒指/耳饰/摆件
   - **价格**：输入成本价和售价（可点击"定价建议"获取参考价）
   - **规格**：根据器型自动显示对应规格字段（如手镯显示圈口、戒指显示尺码）
   - **标签**：选择种水（冰种/糯种等）、颜色（满绿/飘花等）、工艺、题材标签
   - **图片**：上传货品照片，可设封面图
   - **柜台**：选择所在柜台编号
   - **证书号**：如有鉴定证书可录入
4. 点击"确认"保存

### 2.3 通货入库流程（批次采购）

**适用场景**：同一批次采购多件同类货品（如一批翡翠手镯）

**第一步：创建批次**
1. 切换到 **批次管理** Tab
2. 点击 **"新增批次"** 按钮
3. 填写：批次号（如 FC-20260401-001）、材质、器型、总数量、总成本、分摊方式、供应商、采购日期
4. 确认创建

**第二步：逐件录入货品**
1. 在批次卡片中点击 **"快速录入"**
2. 系统自动关联批次信息（材质、供应商、采购日期）
3. 每件货品仅需填写：售价、柜台、规格
4. 录入完成后，批次进度条更新

**第三步：成本分摊**
1. 所有货品录入完成后，批次进度达到100%
2. 点击批次卡片的 **"分摊"** 按钮
3. 系统按选定方式分配成本：
   - **均摊**：总成本 ÷ 数量
   - **按克重**：按重量比例分配
   - **按售价**：按售价比例分配
4. 确认分摊，每件货品的 allocatedCost 更新

### 2.4 销售流程

**单品销售**：
1. 切换到 **库存管理** Tab
2. 在搜索框输入 SKU 编码，或点击扫码按钮扫描条码
3. 系统自动弹出销售对话框
4. 填写：实际售价、销售渠道（门店/微信）、客户（可选）、支付方式、支付状态
5. 确认销售，货品状态变为"已售"

**套装销售**（多件组合）：
1. 在库存管理中勾选多件要组合销售的货品
2. 点击"批量售卖" → 选择"套装销售"
3. 输入套装总价
4. 选择分摊方式：
   - **按比例**：按各件售价比例分配
   - **链扣成本**：配链按成本价，剩余归主石
5. 确认后每件货品生成独立销售记录

### 2.5 退货流程

1. 切换到 **销售记录** Tab
2. 找到需要退货的销售记录
3. 点击 **"退货"** 按钮
4. 填写：退款金额、退货原因（质量/尺码/后悔/其他）、退货日期
5. 确认退货，货品状态变为"已退货"

### 2.6 贵金属调价流程

**适用场景**：黄金/18K金等贵金属市价变动

1. 切换到 **系统设置** → **贵金属市价** 子Tab
2. 选择要调价的贵金属材质
3. 输入新的每克价格
4. 点击 **"预览影响"**：系统显示所有关联货品的旧价/新价对比
5. 确认调价：所有关联货品的成本价和底价自动更新
6. 调价历史可在"价格历史"中查看

## 3. 各模块操作指南

### 3.1 利润看板

**进入方式**：登录后默认展示，或按数字键 `1`

**主要功能**：
- **时段筛选**：右上角选择 月份/季度/年度/全部/自定义日期范围
- **KPI卡片**：显示库存量、月营收、月利润、压货预警数、回本批次数
- **迷你KPI**：库存总值、平均成本、周转率、毛利率等
- **图表区域**：向下滚动查看各类分析图表（趋势/分布/排行）
- **手动刷新**：点击右上角刷新按钮

**快速跳转**：点击库存分布图表中的分类可直接跳转到库存管理页面对应筛选

### 3.2 库存管理

**进入方式**：数字键 `2` 或点击导航栏"库存"

**筛选操作**：
1. 页面顶部筛选区域提供多维度筛选
2. 先选状态（在库/已售/已退），再选材质大类→具体材质
3. 可叠加柜台、批次、价格区间、入库日期范围筛选
4. 搜索框支持按SKU、名称、证书号搜索
5. 已激活的筛选条件以标签形式显示，点击可快速移除

**批量操作**：
1. 勾选多个货品（表头全选或逐行勾选）
2. 底部出现批量操作栏：批量售卖/调价/删除/换柜/打印标签
3. 批量调价支持预览：输入比例或金额后显示调价前后对比

**行内编辑**：点击售价单元格可直接修改，按回车保存

**快捷键**：
- `Ctrl+K`：聚焦搜索框
- `Ctrl+N`：新增货品
- `Ctrl+E`：导出数据

### 3.3 销售记录

**进入方式**：数字键 `3` 或点击导航栏"销售"

**查看销售**：
- 顶部显示今日/本周/本月营收统计
- 快捷日期按钮一键切换时间段
- 支持按渠道（门店/微信）和日期范围筛选

**支付管理**：
- 销售记录的支付状态以彩色徽章显示
- 点击支付状态徽章可直接修改支付方式和状态
- 五种状态：已付(绿)/待付(黄)/部分付(橙)/逾期(红)

**打印小票**：点击销售记录的打印按钮，生成可打印的小票页面

### 3.4 批次管理

**进入方式**：数字键 `4` 或点击导航栏"批次"

**查看批次**：
- 卡片式展示所有批次
- 每张卡片显示：批次号、材质、总成本、录入进度条、已售数量、营收、利润率、回本率
- ROI排行榜展示利润率最高的5个批次

**成本分摊条件**：批次录入进度必须达到100%，且批次内无已售货品

### 3.5 客户管理

**进入方式**：数字键 `5` 或点击导航栏"客户"

**客户详情**：
- 点击客户行展开详情面板
- 消费统计：累计消费、订单数、平均客单价
- 月度消费柱状图
- 偏好分析：最常购买的材质
- 购买时间线：可视化展示每笔购买和退货记录

**VIP等级**：
- 普通：累计消费 0-5,000 元
- 银卡：5,000-20,000 元
- 金卡：20,000-50,000 元
- 钻石：50,000 元以上
- 每个客户卡片显示当前等级和距下一等级的进度

**标签系统**：可为客户添加自定义标签（如"翡翠收藏家"、"VIP"、"婚戒客户"），支持按标签筛选

### 3.6 操作日志

**进入方式**：数字键 `7` 或点击导航栏"日志"

- 自动记录所有增删改操作
- 不同操作类型用颜色区分（创建=绿、编辑=蓝、删除=红、销售=紫、退货=橙）
- 支持按操作类型和日期范围筛选
- 可开启10秒自动刷新

### 3.7 系统设置

**进入方式**：数字键 `6` 或点击导航栏"设置"

**6个子Tab**：

1. **字典管理**：管理材质（5大类36种）、器型（9种）、标签（4组20个），支持启用/停用
2. **贵金属市价**：设置克价→预览影响→确认调价→查看历史
3. **供应商管理**：维护供应商联系信息
4. **系统配置**：经营成本率、上浮比例、压货预警天数等KV配置
5. **数据备份**：下载.db文件备份 / 上传备份恢复数据库
6. **数据导入**：支持JSON和CSV格式导入货品/销售数据

**数据备份最佳实践**：
- 建议每周下载一次备份
- 恢复前系统自动保存当前数据库副本
- 备份文件为 SQLite .db 格式，可用任何SQLite工具打开

## 4. 移动端操作指南

系统自动适配手机和平板屏幕：

- **底部导航栏**：固定在页面底部，快速切换功能模块
- **快捷操作按钮(FAB)**：右下角浮动按钮，快速新增货品/销售/批次
- **底部统计栏**：显示在库数/今日销售/今日营收
- **卡片视图**：货品列表自动切换为卡片式布局
- **扫码功能**：点击搜索框旁的相机图标启用摄像头扫码

## 5. 数据导入导出

### 5.1 导出数据

在任何列表页面（库存/销售/批次/客户/日志），点击"导出"按钮即可导出当前筛选结果为CSV文件。

CSV文件采用UTF-8 BOM编码，可直接用Excel打开，中文不乱码。

### 5.2 导入数据

1. 切换到 **系统设置** → **数据导入** 子Tab
2. 选择导入类型（货品/销售）
3. 上传JSON或CSV文件
4. CSV导入流程：
   - 系统自动检测列名并映射
   - 可手动调整列映射关系
   - 预览校验结果（必填字段缺失、格式错误等）
   - 确认导入，显示进度条
5. 可下载CSV模板按格式填写

## 6. 常见问题

**Q: 忘记密码怎么办？**  
A: 通过环境变量 `ADMIN_PASSWORD` 重新设置，或删除数据库文件重新初始化（会丢失所有数据）。

**Q: 数据存储在哪里？**  
A: 数据存储在 SQLite 数据库文件 `/app/db/custom.db`，Docker部署时已挂载为持久卷，容器重建不丢失。

**Q: 支持多用户同时使用吗？**  
A: 当前为单用户设计，仅支持一个管理员账号。多设备可同时登录同一账号。

**Q: 断网后还能用吗？**  
A: 断网后系统会显示提示横幅，但已加载的页面仍可操作。网络恢复后自动重连。

**Q: 如何更换主题？**  
A: 点击导航栏右侧的主题切换按钮，支持浅色/深色/跟随系统三种模式。

**Q: 图片存储在哪里？**  
A: 图片存储在服务端 `/app/upload/` 目录，Docker部署时挂载为持久卷。

**Q: 如何升级系统？**  
A: Docker方式：拉取最新镜像后重新创建容器即可，数据卷自动保留。
```bash
docker pull gg480/jade-inventory-next:latest
docker compose down && docker compose up -d
```

**Q: 支持什么浏览器？**  
A: 推荐使用 Chrome / Edge / Safari 最新版本。移动端支持 iOS Safari 和 Android Chrome。
