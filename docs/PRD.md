## 1. 产品概述

### 1.1 产品定位

翡翠珠宝进销存管理系统（Jade Inventory）是一款面向中小型翡翠珠宝零售门店的全功能业务管理工具。系统覆盖从采购入库、库存管理、销售记录、客户管理到数据分析的完整业务链，帮助店主实现数字化经营、精细化管理。

### 1.2 目标用户

- 翡翠、和田玉、水晶、贵金属等珠宝零售门店经营者
- 拥有1-5个柜台的小型珠宝店铺
- 需要同时管理门店销售和微信渠道销售的商户
- 对库存周转、利润分析有明确需求的珠宝从业者

### 1.3 核心价值

| 价值维度 | 描述 |
|---------|------|
| **效率提升** | 扫码/搜索快速开单，批量化操作减少重复劳动 |
| **成本透明** | 批次成本分摊、贵金属市价联动，成本一目了然 |
| **决策支撑** | 多维度看板、利润分析、周转率追踪，数据驱动经营 |
| **客户经营** | VIP分层、消费偏好追踪、回购提醒，提升复购率 |
| **安全可控** | 密码哈希存储、API全链路鉴权、操作日志审计 |

### 1.4 产品形态

- 单用户（店长/店主）使用的 Web 应用
- 自部署（Docker / NAS），数据完全自主
- 移动端适配（响应式设计），支持手机操作
- 离线容错（断网提示，恢复后自动重连）

## 5. 术语表

| 术语 | 说明 |
|------|------|
| SKU | Stock Keeping Unit，库存量单位，每件货品唯一编码 |
| 批次(通货) | 同批采购的货品集合，总成本需分摊到每件 |
| 均摊 | 批次成本平均分配到每件货品 |
| 按克重分摊 | 按重量比例分配成本 |
| 按售价分摊 | 按售价比例分配成本 |
| 压货预警 | 在库超过N天未售出的货品标记 |
| 回本率 | 批次已售营收 / 批次总成本 |
| 套装销售 | 多件货品合并销售，总价按规则分摊 |
| VIP分层 | 按累计消费额自动分级 |
| 调价 | 贵金属市价变更后联动更新相关货品成本 |
